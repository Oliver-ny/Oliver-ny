🐯基于跳表的轻量级键值型存储引擎 （2023年06月-2023年07月）

项目描述

该项目使用C++语言实现了轻量级键值对存储引擎，其底层数据结构为跳表，支持数据随机增删查改，以及数据的落盘加载功能。

项目细节

  • 使用模板编程实现跳表存储结构，实现插入数据、删除数据、查询数据、数据展示、数据落盘、文件加载数据和数据库大小展示功能
  • 使用锁机制保证读写分离，每秒支持十万次以上随机读写

实施细节

模板类

主体包含两个类：结点类（Node）、跳表类 （SkipList）

Node类的设计：

  • 数据：
    • K key 键
    • V value 值
    • Node<K, V>** forward 指向Node*数组的指针（指向的是一个数组，数组中存放的是下一个Node结点的指针，数组的下标表示是第几层，举个栗子，this->forward[0]代表了指向当前Node结点在第0层上下一个结点的指针
    • int node_level 用于指定forward所指向的数组的大小，该结点存在的层数
  • 函数：
    • 构造函数
    • 析构函数
    • 获取key的函数（用const修饰）
    • 获取value的函数（用const修饰）
    • 设置value的函数

SkipList类的设计

  • 数据：
    • int max_level 定义跳表最大的层数
    • int skip_list_level 当前所使用的层数（因为当前所使用的层数可能还没有到最大的层数的限制）
    • Node<K, V>* header 头结点（我更倾向于叫其dummynode）
    • int element_count跳表的元素个数
    • ofstream file_writer 用于向文件中写数据
    • ifstream file_reader 用于从文件中读数据
  • 函数：
    • 构造函数
    • 析构函数
    • 创造结点Node的函数
    • 向跳表添加元素
    • 从跳表中删除元素
    • 在跳表中查询元素
    • 将跳表所有元素保存到文件中
    • 从文件中读取数据放到跳表中
    • 获取随机的层数
    • 打印跳表的函数
    • 获取跳表元素个数的函数

其中，重点是添加、删除、查询元素

添加元素的实现过程：

  1. 打开互斥锁
  2. 定义一个update数组（存的是Node*，大小为最大层数）
  3. 从当前所使用的层数开始，寻找需要插入的位置，把位置需要插入的Node结点的指针存到update数组中
  4. 在第0层查找是否存在需要插入的健，如果存在提醒并返回（注意解锁）
  5. 如果不存在，则执行以下的插入步骤

a. 调用随机获取层数的函数得到需要随机插入的层数（每层都插入的话数据量增大的话，跳表就没什么作用了

b. 如果随机生成的层数大于当前所使用的层数，那么将大于当前层数的update数组的值均设为head，当前所使用的层数也需要更新

c.从第0层开始到random_level层，依次链接新节点

  1. 跳表元素量加1
  2. 解开互斥锁

删除元素的实现过程和添加元素类似，先查找到需要删除的元素的位置，再从第0层开始依次删除（当前层不存在就跳出循环，因为当前层不存在的话，那么上一层也一定不存在（同样记得要上互斥锁）

查找元素相对来说更加简单，可以看作是添加和删除的基础

获取随机的层数这个函数是利用概率的思想来实现的，随机生成数字，如果当前数字是奇数，那么层数++，一直循环，直到当前生成的随机数是偶数为止（层数的初始值为1，那么层数为2的概率为1/2，依次类推可以得到随机生成的层数越大的概率是越小的，这样可以有效的避免后期跳表所存储的数据量越来越大所带来的效率下降的问题，因此这个函数很重要

🐯基于多源信息融合的坐卧式智能康复机器人关键技术研究（2020年08月-2022年12月）

项目描述

针对下肢康复训练场景中三维人体位姿估计精度易受人机交互、遮挡等因素影响的问题，提出了一种精度高、非接触式、消费者级别的三维人体下肢位姿估计方法。

项目细节

  • 搭建实验场景，构建运动捕捉系统及多相机系统，完成多系统同步控制（基于Redis）及多系统标定的功能，并在此基础上完成数据集采集。
  • 方法设计，基于计算机视觉，使用多视角几何，以立体图像为输入，通过添加人体内在的生理约束以及人与环境交互过程中的物理约束，有效提高了位姿估计的精度。
  • 进一步优化，针对视觉方法对自遮挡和相互遮挡的局限，通过引入惯性传感器与相机多源信息融合方法，提升了本方法对于遮挡情形下的鲁棒性。

实施细节

实验场景搭建

  • 运动捕捉系统

采用OptiTrack提供的硬件及配套软件

  • 多相机系统

一台Kinect+两台Realsense作为多相机系统

多系统同步控制

基于Qt设计GUI用于同步控制多系统，实现逻辑基于Redis非关系型数据库。通过按钮去控制同时启动与结束不同系统的数据采集，并以时间戳的格式去存储不同系统所捕获的信息，便于后续数据的处理。

多系统标定

  • 运动捕捉系统标定

使用商业化软件，只需在场地中心不断移动特制的标定杆，待软件提示后即可自动求解标定信息，将未标定的多个运动捕捉相机组合成为一个标定好的运动捕捉系统，之后在场地中放置特制的标定板指定坐标系原点后即完成了运动捕捉系统的标定。

  • 运动捕捉系统与多相机系统联合标定

使用特殊的自制棋盘标定板，因为根据“张正友标定法”利用普通的棋盘标定板只能标定多相机系统中多个相机的内参及外参，并不能标定出多相机系统坐标系与运动捕捉系统坐标系之间的关系。

在棋盘四个角点处贴上特制的反光标定点，使标定板能够同时被运动捕捉系统和多相机系统识别，进而编写程序通过两个系统中棋盘位置的信息来计算两个系统坐标系的转换关系。

数据采集

分为定量数据采集和定性数据采集两个部分。定量数据采集要求被试者穿上特定服装贴上反光标记点，使用运动捕捉系统捕获运动的真值。定性数据则没有要求，正常采集即可。

方法设计

（论文尚在发表阶段，仅大致描述，后续接收后详细补充）

分为三个阶段，第一阶段，基于多视角几何的三维位姿求解；第二阶段，基于物理、生理等约束信息的三维位姿精化；第三阶段，引入惯性信息作为约束进行进一步细化

基于多视角几何的三维位姿求解

  • 二维姿态检测

结合现有先进的姿态检测器从多视图图像中提取出二维姿态信息

  • 三维位姿求解

基于三角测量原理，利用检测到的二维姿态及相机内外参联立方程，求解三维位姿

基于物理、生理等约束信息的三维位姿精化

  • 下肢模型定义

从简易实用的角度出发，过于精细复杂的模型会加重优化的计算量拖慢优化速度，过于简单的模型不能很好的表现下肢活动。

  • 约束建模

    • 物理约束

      下肢康复训练的过程中，患者会与康复器材产生频繁的交互，这些物理上的交互关系会极大地限制人体位姿。因此，可利用已知的坐垫平面信息限制人体臀部关节的位置，将人体臀部建模为弹簧，进行精准而有意义的位姿估计

    • 生理约束

      • 骨长约束

        通过限制下肢模型关节间的距离，确保骨长在优化的过程中在合理范围内波动，避免骨长的突变

      • 关节角度约束

        限制优化过程中关节的活动角度，避免不合理姿势的出现

基于惯性信息的三维位姿精化

由于现有视觉方法对自遮挡和相互遮挡存在一定的局限性，可以在下肢佩戴几个惯性传感器采集惯性信息，对视觉方法估计出来的结果进行进一步细化。
